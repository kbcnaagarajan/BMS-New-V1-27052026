# Focused Operational Workflows

## 1. Super Admin Creates Company

Super Admin Login
-> Open Platform Dashboard
-> Open Companies
-> Create Company
-> Enter Company Details
-> Select Package / Subscription
-> Enter Company Admin Name and Email
-> Save Company
-> System Creates Company Record
-> System Generates Invite Token
-> Company Admin Accepts Invite
-> Company Becomes Active
-> Super Admin Sees Company Summary Only

## 5. RBAC and Data Scope Check

User Login
-> System Identifies User Type
-> System Identifies Company
-> System Loads Role Permissions
-> System Applies Company Scope
-> System Applies Project / Self Scope
-> User Opens Menu
-> System Shows Only Allowed Modules
-> User Tries Restricted Module
-> System Denies or Hides Access

Priority validation:

Super Admin
-> Should see companies and platform summary
-> Should not manage tenant staff operational data

Company Admin
-> Should see all data inside own company
-> Should not see other companies

Project Manager
-> Should see assigned projects and related project data

Employee
-> Should see own tasks, own timesheets, own attendance, own leave, own expenses
